MonoDevelop.Tizen

MonoDevelop add-in which adds support for developing and deploying LGPLv2-compliant applications using Mono for Tizen.

Installation

  • Open MonoDevelop or Xamarin Studio.
  • Register the MonoTizen add-in repository (this step will go away soon)
    • Open the add-in manager with Xamarin Studio | Add-in Manager ...
    • Open the Gallery tab
    • Click on the drop-box arrow on the right hand side of "All repositories"
    • Choose Manage Repositories
    • Click on the Add button.
    • Enter http://kitsilanosoftware.github.io/MonoTizen/add-in/ in the Register an on-line repository text field, then press OK.
    • You will be returned to the Add-in Repository Management dialog. Press Close.
  • Install and enable the Tizen development add-in
    • In the Add-in Manager dialog, you will see a Mobile Development tree-view container. Click on it to expand it.
    • You will see the Tizen development plugin. Click on it, then click on Install... and then the Install button again on the modal dialog which appears.
    • You will see the installation proceeding. It only takes a few seconds.
    • In the right hand panel you will now see an Enable button. Click it and icon in the left hand panel will switch from Gray to Green.
    • Close the Add-in Manager dialog.
  • Xamarin Studio | Check for Updates... will automatically pick up any future changes to the add-in.

Usage

Create a new solution of type C# | Tizen | Tizen Native Project.

Before the first build, the Tizen add-in will prompt you for the following:

  • SDK Installation Folder (mandatory): The root of your Tizen SDK installation, the SDB tool should be located at `tools/sdb` or `tools/sdb.exe` relative to the chosen value;
  • Mono Runtime Bundle (mandatory): The path to a ZIP re-packaging of the MonoTizen RPMs. You can download ready to use bundles or build your own from RPMs using these build scripts.
  • Author Key (mandatory): The *.p12 key to use for generating native (`*.tpk`) packages;
  • Author Key Password (mandatory): Password of the *.p12 key;
  • Device ID (optional): The ID of the target device, as provided by sdb devices; if left empty, the native tools will use the default device.
Dialog box can also be accessed via the Project | Tizen SDK Settings... menu item.
